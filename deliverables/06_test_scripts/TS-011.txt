TEST SCRIPT: TS-011
Test Scenario: Register New Account
User Story: As a Visitor, I want to create a new buyer account with my details, so that I can make purchases on the website
Priority: Critical
Related Requirements: FR-002

GIVEN the user is on the ecommerce website
  AND the user is not logged in
  AND the user clicks on "Register" or "Sign Up"

WHEN the registration form is displayed
  AND the user enters valid registration data:
    - First Name: "Emily"
    - Last Name: "Johnson"
    - Email: "emily.johnson@email.com"
    - Contact Number: "555-0101"
    - Password: "Pass@123"
    - Confirm Password: "Pass@123"
  AND the user checks "Accept Terms and Conditions"
  AND the user clicks "Submit" or "Register"

THEN the system should validate all input fields
  AND verify the email is unique (not already registered)
  AND verify password meets complexity requirements
  AND verify password and confirm password match
  AND create a new user account with status "Unverified"
  AND generate an email verification token
  AND send a verification email to "emily.johnson@email.com"
  AND display a success message: "Registration successful. Please check your email to verify your account."

WHEN the user checks their email inbox
  AND clicks on the verification link in the email

THEN the system should validate the verification token
  AND update the account status to "Verified"
  AND display a confirmation message: "Email verified successfully. You can now log in."
  AND redirect the user to the login page

WHEN the user attempts to register with an already registered email

THEN the system should display an error message: "This email is already registered. Please login or use a different email."
  AND not create a duplicate account

EXPECTED RESULT:
- New user account is successfully created
- Email verification link is sent to the user
- Email verification completes successfully
- User can log in after email verification
- Duplicate email registration is prevented
- All validation rules are enforced
