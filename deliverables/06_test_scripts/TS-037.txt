TEST SCRIPT: TS-037
Test Scenario: View Order History
User Story: As a Logged-in Buyer, I want to see a list of all my past and current orders, so that I can track my purchase history
Priority: High
Related Requirements: FR-012

GIVEN the user is logged in as "emily.johnson@email.com"
  AND the user has placed 3 orders:
    - Order #1001: Date 2024-11-01, Total $94.48, Status "Delivered"
    - Order #1002: Date 2024-11-05, Total $129.99, Status "Shipped"
    - Order #1003: Date 2024-11-10, Total $59.98, Status "Confirmed"

WHEN the user navigates to "My Account"
  AND clicks on "My Orders" or "Order History"

THEN the system should display a list of all orders for the user
  AND orders should be sorted by date (most recent first)
  AND each order in the list should show:
    - Order Number
    - Order Date
    - Order Total
    - Order Status
    - Number of items
    - "View Details" button/link

WHEN the user clicks "View Details" on Order #1002

THEN the system should display the order detail page showing:
  - Order Number: #1002
  - Order Date: 2024-11-05
  - Current Status: "Shipped"
  - Items ordered:
    - Product name, size, color
    - Quantity
    - Unit price
    - Subtotal
  - Pricing breakdown:
    - Items total
    - Shipping cost
    - Tax
    - Order total
  - Billing address
  - Shipping address
  - Payment method used (last 4 digits of card)
  - Shipment tracking information:
    - Carrier name
    - Tracking number
    - "Track Order" link/button

WHEN the user has no order history

THEN the system should display a message: "You haven't placed any orders yet"
  AND provide a link: "Start Shopping"

EXPECTED RESULT:
- All user orders are displayed correctly
- Order information is accurate and complete
- Orders are properly sorted by date
- Order details page shows comprehensive information
- Empty order history is handled gracefully
- Tracking information is accessible for shipped orders
