TEST SCRIPT: TS-026
Test Scenario: Proceed to Checkout
User Story: As a Logged-in Buyer, I want to initiate the checkout process from my shopping cart, so that I can complete my purchase
Priority: Critical
Related Requirements: FR-008

GIVEN the user is logged in as "emily.johnson@email.com"
  AND the user's shopping cart contains:
    - Product 1: Classic Cotton Shirt, Size M, Color Red, Qty 2, Price $29.99 each
    - Product 2: Casual T-Shirt, Size L, Color Blue, Qty 1, Price $19.99
  AND the cart subtotal is $79.97

WHEN the user navigates to the shopping cart page
  AND reviews the cart items
  AND clicks "Proceed to Checkout" button

THEN the system should validate the cart is not empty
  AND validate all products are still in stock
  AND validate all products are still active
  AND redirect the user to the checkout page

WHEN the checkout page loads

THEN the system should display:
  - Checkout progress indicator (Step 1: Shipping Address)
  - Order summary sidebar showing:
    - List of items with quantities and prices
    - Subtotal: $79.97
    - Shipping: (to be calculated)
    - Tax: (to be calculated)
    - Order Total: (to be calculated after address)
  - Billing address section with:
    - Saved addresses (if any)
    - Option to add new address
    - Form fields: Street, City, State, ZIP, Country
  - Shipping address section with:
    - Checkbox: "Same as billing address"
    - Option to select different address
    - Form fields if different address selected

WHEN the user has an empty cart and tries to access checkout directly via URL

THEN the system should redirect to the shopping cart page
  AND display a message: "Your cart is empty. Please add items before checkout."

WHEN a product in the cart becomes out of stock before checkout

THEN the system should display a warning: "Some items in your cart are no longer available"
  AND highlight the unavailable items
  AND prevent checkout until items are removed

EXPECTED RESULT:
- User can successfully proceed to checkout with valid cart
- Checkout page displays all necessary information
- Order summary is accurate
- Empty cart checkout is prevented
- Out of stock items are handled appropriately
- User can navigate through checkout steps
