TEST SCRIPT: TS-080
Test Scenario: Create New Product
User Story: As a Logged-in Admin, I want to add a new product with all details, so that I can make it available for purchase
Priority: Critical
Related Requirements: FR-019

GIVEN the admin is logged in to the admin panel
  AND categories exist in the system:
    - "Shirts" (category)
    - "Casual Shirts" (sub-category under Shirts)
    - "Formal Shirts" (sub-category under Shirts)
  AND the admin navigates to "Products" â†’ "Add New Product"

WHEN the product creation form is displayed

THEN the system should show form fields:
  - Product Name* (text)
  - SKU* (text, unique)
  - Category* (dropdown)
  - Sub-Category (dropdown, dependent on category)
  - Description* (rich text editor)
  - Price* (USD, decimal)
  - Keywords (text, comma-separated)
  - Available Sizes* (multi-select: XS, S, M, L, XL, XXL)
  - Available Colors* (multi-select or color picker)
  - Thumbnail Image* (file upload)
  - Product Images (multiple file upload)
  - Status* (Active/Inactive)
  - Meta fields (optional)

WHEN the admin enters valid product information:
  - Product Name: "Premium Cotton Polo Shirt"
  - SKU: "APP-PLO-042"
  - Category: "Shirts"
  - Sub-Category: "Casual Shirts"
  - Description: "High-quality cotton polo shirt perfect for casual occasions..."
  - Price: "39.99"
  - Keywords: "polo, shirt, casual, cotton, comfortable"
  - Sizes: [S, M, L, XL]
  - Colors: [Navy Blue, White, Black, Red]
  - Thumbnail: "polo-shirt-main.jpg"
  - Additional Images: ["polo-blue.jpg", "polo-white.jpg", "polo-black.jpg"]
  - Status: "Active"
  AND clicks "Save Product" or "Publish"

THEN the system should validate all required fields are filled
  AND verify SKU "APP-PLO-042" is unique (not already used)
  AND verify price is a valid positive number
  AND verify at least one size is selected
  AND verify at least one color is selected
  AND verify thumbnail image is uploaded
  AND upload all images to the server/storage
  AND create a new product record in the database
  AND display success message: "Product created successfully"
  AND redirect to the product list or product detail page
  AND the new product should appear in the product catalog

WHEN a buyer searches for or browses the catalog

THEN the new product "Premium Cotton Polo Shirt" should be visible
  AND available for purchase if status is "Active"
  AND display all entered information correctly

WHEN the admin attempts to create a product with duplicate SKU

THEN the system should display validation error: "SKU already exists. Please use a unique SKU"
  AND not create the product
  AND highlight the SKU field

WHEN the admin submits the form with missing required fields

THEN the system should display validation messages for all missing fields:
  - "Product name is required"
  - "SKU is required"
  - "Category is required"
  - "Price is required"
  - etc.
  AND not create the product
  AND remain on the form with entered data preserved

EXPECTED RESULT:
- New product is successfully created with all details
- All product information is saved correctly
- Images are uploaded and associated with the product
- SKU uniqueness is enforced
- Product is immediately available in the catalog (if Active)
- All validation rules are enforced
- Admin receives clear feedback on success or errors
