TEST SCRIPT: TS-053
Test Scenario: Admin Login
User Story: As an Admin User, I want to log into the admin panel with username and password, so that I can manage the ecommerce website
Priority: Critical
Related Requirements: FR-014

GIVEN there is an admin account with:
  - Username: "admin@ecommerce.com"
  - Password: "Admin@Secure123"
  - Role: "Admin"
  - Status: "Active"
  AND the admin is not currently logged in

WHEN the admin navigates to the admin panel URL (/admin)
  AND the admin login page is displayed

THEN the system should show:
  - Username/Email field
  - Password field
  - "Login" button
  - "Forgot Password" link

WHEN the admin enters username "admin@ecommerce.com"
  AND enters password "Admin@Secure123"
  AND clicks "Login" button

THEN the system should validate the credentials
  AND verify the username exists in admin users table
  AND verify the password matches
  AND verify the account status is "Active"
  AND create an admin session
  AND redirect to the admin dashboard
  AND display admin dashboard showing:
    - Total registered buyers (active/inactive)
    - Total products count
    - Revenue statistics (today/this month)
    - Recent orders
    - Navigation menu with admin functions

WHEN invalid credentials are entered

THEN the system should display error: "Invalid username or password"
  AND not create a session
  AND remain on login page
  AND allow retry

WHEN the admin account status is "Inactive"

THEN the system should display error: "Account is inactive. Please contact system administrator"
  AND not create a session

EXPECTED RESULT:
- Admin can successfully log in with valid credentials
- Admin dashboard loads correctly after login
- Invalid credentials are rejected
- Inactive accounts cannot log in
- Session is created and persists
- Admin has access to all administrative functions
